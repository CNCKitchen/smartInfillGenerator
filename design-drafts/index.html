Smart Infill Generator — three design drafts

Static mockups, all showing the same state (step 5 just converged on bracket_v3.stl) so the comparison is fair. All three share the reorganized layout: top app bar, numbered step rail, one active-step panel, dominant viewport, status bar with telemetry. Open at ≥ 1280 px width.

Draft 1 · Drawing Officelight

The engineering drawing as interface: sheet frame around the whole app, a real title block as the header, density regions rendered as section hatching with dimension-style leader annotations, line-weight hierarchy instead of shadows.
  • Accent: checker's red — spent only on the active step, primary actions, loads
  • Type: Archivo + IBM Plex Mono (every number is mono)
  • Signature: title block + hatched regions + leader-line callouts

Draft 2 · Werkbanklight

Software as a measuring instrument: warm-gray chassis with recessed input wells, a caliper-scale step rail with an orange carriage, results as DRO-style readout windows, machine status strip with a green ready-lamp. CNC Kitchen orange as the accent.
  • Accent: anodized orange (brand) + LCD-green readouts
  • Type: Barlow (DIN heritage) + B612 Mono (the Airbus cockpit face)
  • Signature: caliper rail + digital-readout result cards

Draft 3 · Operatordark

Viewport-first operator console in warm graphite (deliberately not blue-black): the "log for nerds" promoted to a first-class telemetry dock with scope-style amber traces, floating instrument readouts over the stage, minimal chrome.
  • Accent: instrument amber on layered graphite — no neon, no gradients
  • Type: Chivo + Fragment Mono
  • Signature: the telemetry dock (charts + live log) under the viewport

Drafts only — controls are not wired. Fonts load from Google Fonts (OFL-licensed; they would be self-hosted in the real app). Pick a direction and it gets implemented as the actual React/CSS theme.
